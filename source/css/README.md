# CSS 文件架构说明

## 文件结构

本主题的CSS文件架构如下：

### 主要CSS文件

- **`typing-desktop-only.css`** - 纯桌面端样式文件（当前默认使用）
  - 强制所有设备显示PC端布局
  - 完全移除所有移动端适配和响应式样式
  - 使用!important确保样式优先级
  - 确保移动端导航完全隐藏，PC端导航始终显示
  - 适合需要统一PC端体验的场景

- **`typing-pc.css`** - PC端专用样式文件（备用）
  - 包含所有基础样式和桌面端布局
  - 包含主题色彩方案、字体设置、暗色模式样式
  - 包含代码高亮样式
  - 不包含移动端响应式样式，确保PC端体验统一

- **`typing.css`** - 完整版样式文件（备用）
  - 包含所有基础样式、桌面端布局和完整的响应式设计
  - 包含移动端响应式样式
  - 如需恢复完整的响应式功能，可将head.ejs中的`typing-desktop-only`改为`typing`

- **`typing-mobile.css`** - 移动端样式补充文件
  - 包含关键的移动端响应式样式调整
  - 支持设备宽度：768px, 480px, 320px
  - 包含移动端导航菜单样式
  - 用于选择性加载移动端样式

### 备份文件

- **`typing.css.original`** - 原始文件备份
  - 完整的原始CSS文件备份
  - 如需恢复可以重命名此文件

- **`typing.css.backup`** - 临时备份文件

## 加载方式

主题通过 `layout/_partial/head.ejs` 文件加载：

```html
<%- css('css/typing-desktop-only') %>
<!-- PC端专用样式 - 强制所有设备显示桌面端布局 -->
<!-- 移动端样式已完全禁用 -->
<%- css('css/donate') %>
```

## 架构优势

1. **更好的可维护性**：PC端和移动端样式分离，便于独立维护
2. **按需加载**：可以根据需要选择性加载样式文件
3. **代码可读性**：文件职责清晰，便于理解和修改
4. **主题定制**：方便后续的主题定制和扩展
5. **调试便利**：可以针对特定设备进行样式调试

## 使用说明

### 如需启用移动端样式

如果需要重新启用移动端适配，可以：

1. 修改 `layout/_partial/head.ejs` 文件
2. 取消注释移动端样式文件的加载：
   ```html
   <%- css('css/typing-mobile') %>
   ```

### 如需恢复完整的响应式功能

如果需要恢复到完整的响应式设计（推荐）：

1. 修改 `layout/_partial/head.ejs` 文件
2. 将 `typing-pc` 改为 `typing`：
   ```html
   <%- css('css/typing') %>
   <%- css('css/typing-mobile') %>
   ```

### 如需恢复原始版本

如果需要恢复到原始的单文件CSS架构：

1. 删除或重命名当前的 `typing.css`
2. 将 `typing.css.original` 重命名为 `typing.css`
3. 修改 `layout/_partial/head.ejs` 中的CSS引用

### 自定义样式

- **PC端自定义**：在 `typing-desktop.css` 文件末尾添加自定义样式
- **移动端自定义**：在 `typing-mobile.css` 文件末尾添加自定义样式
- **响应式断点**：可以根据需要添加新的媒体查询

## 技术规格

- **字符编码**：UTF-8
- **CSS语法**：CSS3
- **兼容性**：现代浏览器（IE10+, Chrome, Firefox, Safari, Edge）
- **响应式断点**：
  - 1024px+：桌面端
  - 769px-1024px：平板端
  - ≤768px：移动端
  - ≤480px：小屏手机
  - ≤375px：窄屏手机
  - ≤360px：超小屏手机
  - ≤320px：极致小屏
